## 二、个人毕业设计（论文）工作体会

### 1. 工作量与任务强度
本课题工作量总体较大，属于“算法研究 + 工程实现 + 实验分析”三线并行。除了实现单个检测算法，还需要完成多算法统一接口、前后端联调、指标体系构建、模型训练与结果可视化，任务跨度覆盖了计算机视觉、深度学习、后端服务和前端交互四个方向。

### 2. 技术难度与主要挑战
1. 算法层面的难点
- 传统视觉方法对图像质量和参数敏感，面对噪声、边缘断裂、孔洞形态不规则时易波动。
- 深度学习方法虽然鲁棒性更强，但需要较高质量数据与规范训练流程，且工程部署复杂度更高。

2. 工程层面的难点
- 多算法结果统一输出与公平比较并不简单，需要统一字段、统一可视化和分算法指标。
- 推理效率与系统稳定性之间存在平衡问题，例如模型加载开销、前后端通信格式、图像编码开销等。

### 3. 研究方法与实施路径
本课题采用“传统方法 + 深度学习方法”并行对照的研究思路：
- 先完成传统方法，建立可解释、可调参数的基线；
- 再引入 YOLO 分割模型，提高复杂场景下的鲁棒性；
- 最后通过统一评价指标体系进行对比分析，实现定量结论。

同时采用“边开发边验证”的工程方法：每完成一个模块立即接入系统，减少后期集成风险。

### 4. 当前效果与阶段结论
- 系统已经能够稳定完成单图检测和四算法对比。
- 指标计算链路完整，可对同一图像给出可量化评价结果。
- YOLO 模型在当前数据集上取得较高精度，说明方案具备可行性。

### 5. 特色亮点
1. 统一检测框架
- 将传统视觉算法与深度学习算法统一到同一接口和同一展示平台，便于对照研究。

2. 分算法指标设计
- 针对不同算法机制设置对应指标，而不是“一把尺子量到底”，增强结论可信度。

3. 工程闭环完整
- 已实现“数据 -> 训练 -> 推理 -> 可视化 -> 导出”的完整闭环，具备展示和复现实验能力。

## 三、存在的问题与解决方案

### 问题1：缺少真实场景数据集，数据来源相对单一
问题说明：
- 当前数据集规模可以支撑模型训练与验证，但仍偏向实验条件，真实工况覆盖不够充分（如光照剧变、泥浆遮挡、设备抖动、异形孔口等）。

影响：
- 可能导致模型在真实环境中的泛化能力下降，出现精度回落或误检漏检。

解决方案：
1. 数据补充：
- 与工程现场或实验室合作采集真实图像，优先补充“困难样本”（遮挡、模糊、偏心、强噪声）。
2. 数据治理：
- 建立统一标注规范，进行复核与抽检，减少标签噪声。
3. 训练策略：
- 分阶段训练（先混合训练，再真实数据微调），并增加针对性增强策略。
4. 验证策略：
- 单独建立真实场景测试集，按场景类别统计指标，形成分场景结论。

### 问题2：对部分代码细节理解还不完全，尤其是复杂参数与训练机制
问题说明：
- 能够完成系统搭建与调参，但在部分底层机制上（例如部分超参数耦合关系、评价指标边界条件、模型训练收敛行为）理解仍不够系统。

影响：
- 参数调整时可能偏经验化，难以形成更深入的理论解释；论文论证深度会受到影响。

解决方案：
1. 建立“代码-原理”对照笔记：
- 逐函数梳理输入、输出、关键变量、数学含义，形成个人知识图谱。
2. 小型对照实验：
- 每次只改1-2个关键参数，记录指标变化，建立参数敏感性表。
3. 论文反向驱动学习：
- 以论文章节为任务清单，倒推必须掌握的原理点，边写边补齐理解短板。
4. 导师与同学评审：
- 定期开展代码走查和口头讲解，检验是否真正“讲得清、解释透”。

### 问题3：工程配置仍有待进一步规范化
问题说明：
- 目前已可运行，但在模型路径、依赖声明、实验配置统一化方面仍有提升空间。

解决方案：
1. 统一配置入口：
- 将模型路径、数据路径、阈值参数集中到配置文件，减少硬编码。
2. 环境规范：
- 补全并校验依赖清单，保证跨机器复现。
3. 实验管理：
- 统一实验命名规则和日志归档模板，提升可追溯性。
